hexo建站总结

发表于 2022-05-03 更新于 2022-05-08 分类于 技术 阅读次数：

  1. 最终的使用写作发布方法：

    1. macOS上创建桌面的上的hexo-next-md文件夹
    2. 使用typora编写md文件，并保存到这个文件夹；
    3. 在owncloud中新建一个hexo用户，获得他的webdav地址；
    4. 使用访达连接owncloud容器中hexo用户的webdav文件夹；
    5. 将md文件从mac的hexo-next-md文件夹复制到webdav文件夹中的myhexo/source/_posts子文件夹（或者在编写md文件的时候直接保存到此文件夹）；
    6. 在ECS上运行/root/hexog.sh脚本，它会将webdav子文件夹中的md文件移动到hexo-next容器的md发布文件夹（/mnt/docker/hexo-next/myhexo/source/_posts）中，并远程运行hexo-next容器内的hexo g指令更新网站。
    7. 配置好github deploy，hexo d发布
    8. 在sm.ms注册账号，生成API token;
    9. 使用upic挂载sm.ms图床使用。

    15. container启动自动运行脚本：

      1. 创建一个容器hexo
      2. 在容器内写好希望开机自动执行的脚本hexostart.sh: export PATH=/usr/local/nodejs/bin:$PATH&&cd /root/hexo&&hexo s& &&/bin/bash
      3. 将容器打包成映像hexo:chs
      4. 使用docker run restart=always新映像hexo:chs加上hexostart.sh脚本名称再生成新容器hexo-chs: docker run -dit -v /root/hexo:/root -p 10002:4000 -w /root -h hexo –privileged=true –name=hexo-chs –restart=always hexo:chs /root/hexostart.sh
